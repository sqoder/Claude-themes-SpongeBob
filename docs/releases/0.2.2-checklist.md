# 0.2.2 Release Checklist

Use this checklist for the first fully automated release after `0.2.1`.

## Before tagging

1. Make the code changes for `0.2.2`.
2. Update:
   - `package.json` version
   - `package-lock.json` version
   - `src/patchRuntime.ts` `PATCHER_VERSION`
   - `CHANGELOG.md`
3. Run:

```bash
npm run verify:release
```

4. Commit the release branch or `main` changes:

```bash
git add .
git commit -m "release: 0.2.2"
git push origin main
```

## Create the release

1. Create a GitHub Release with tag:

```text
v0.2.2
```

2. Use a release body based on the latest shipped changes.
3. Publish the GitHub Release.

## Expected automation

After the GitHub Release is published, `.github/workflows/publish.yml` should:

1. verify the release tag matches `package.json`
2. run `npm run verify:release`
3. publish `claude-code-theme-patcher@0.2.2`

## After publish

Run these checks:

```bash
npm view claude-code-theme-patcher version
npm view claude-code-theme-patcher dist-tags --json
npx --yes claude-code-theme-patcher@0.2.2 --help
```

Then confirm:

- GitHub Actions publish workflow is green
- npm package page shows `0.2.2`
- `latest` points to `0.2.2`
- the CLI help output still works through `npx`
